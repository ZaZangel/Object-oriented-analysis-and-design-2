

## Введение

**Цель работы:** реализовать паттерн проектирования Converter (Преобразователь) в приложении для управления заказами клиентов, продемонстрировать преимущества разделения слоёв данных и представления.

**Задачи:**
1. Изучить принципы паттерна Converter и его место в многослойной архитектуре
2. Реализовать классы Entity, DTO, Converter и Repository
3. Обеспечить преобразование данных из модели хранения в модель представления
4. Продемонстрировать работу приложения с графическим интерфейсом
5. Сравнить реализацию с паттерном и без него

**Актуальность:** Паттерн Converter широко применяется в enterprise-разработке для разделения ответственности между слоями приложения, что упрощает тестирование, поддержку и масштабирование кода.


## Глава 1. Теоретическая часть

### 1.1. Паттерн Converter: назначение и принципы

**Converter (Преобразователь)** — это структурный паттерн, который выделяет логику преобразования данных в отдельный класс, изолируя модель хранения (Entity) от модели представления (DTO).

**Ключевые принципы:**
- **Single Responsibility:** каждый класс отвечает только за свою задачу
- **Separation of Concerns:** бизнес-логика отделена от UI и доступа к данным
- **Immutability of DTO:** DTO содержит только данные для отображения, без логики изменения

**Схема потока данных:**
[База данных]
↓
[Repository] → возвращает Entity
↓
[Converter] → преобразует Entity + рассчитывает метрики
↓
[DTO] → готовые данные для отображения
↓
[GUI] → только отрисовка, без расчётов

### 1.2. Архитектурные слои приложения

| Слой | Классы | Ответственность |
|------|--------|----------------|
| **Data Access** | `CustomerRepository` | Работа с БД: CRUD-операции |
| **Domain Model** | `CustomerEntity`, `OrderEntity` | Отражение структуры таблиц БД |
| **Conversion** | `CustomerConverter` | Преобразование Entity → DTO, расчёты |
| **Presentation** | `CustomerDto`, `OrderDto` | Данные для отображения в GUI |
| **UI** | `OrderManagementGui` | Отрисовка интерфейса, обработка событий |
